xmlエディタを作成する。
## 要件
### 概要
- guiを備える
- guiはwebブラウザを使用、またはスタンドアロンアプリとする。
- document_schema.xsdに準拠したxmlを扱う
- xmlのパーサーを使用すること。
- 正規表現など、xmlパーサー以外の方法でタグのとじ開き・整形式担保・妥当性担保の制御をしないこと
- reactとelectronを使用
- 整形式を担保する
- 妥当性検証をする
### 機能
- xmlファイルを書き出せるようにする
- xmlファイルを読み込めるようにする。その際、妥当性検証を行い、妥当でないものは読み込まないようにする
- title1~5とp、tableは、適宜選択して追加できるようにする
- richテキスト型の要素はテキストフィールドにする。
  - インライン要素は、テキストフィールド内の任意のテキストを選択し、右クリックでマークアップできるようにする
  - マークアップの解除も右クリックのメニューに組み込む
  - g：マークアップ後はテキストフィールド内で実際に強調する
  - u：同じく、下線を引く
  - sup：同じく、上付きにする
  - sub：同じく、下つきにする
  - yomikae/ruby/img：属性を持つ。属性の設定の仕方と入力後のguiでの表示の仕方については案を出して。
  - マークアップ時には整形式を担保する。つまり、<u>ア<g>イ</u>ウ</g>といったマークアップはしない。
  - マークアップ削除時にも整形式が崩れないようにする。
- tableについて
  - th/trを追加できるようにする。tr（行）とtd（列）の追加は、いろいろなアプリでよく見かける、あらかじめ行数・列数を指定できるようなものとする
  - 既存のtableについて、行の追加削除・列の追加削除をできるようにする

### 機能（26/4/11作業終了後にユーザが追記）
- GUIのフォントサイズと書体を変更できるようにする。
- GUIの言語設定を変更できるようにする。日英独の3言語。

## 確定事項（ユーザー確認済み）

- 要素名は XSD の `title1~title5` を正とする
- 境界またぎマークアップ：可能なら `<g>A<u>B</u></g><u>C</u>` に分割して適用。不可能（sup/sub/img の境界をまたぐ場合）はエラーメッセージを表示
- `yomikae` / `ruby` / `img` はすべてインライン chip として表示（《よみ》部分の誤削除を防ぐ）
  - `yomikae` / `ruby`：chip 全体がひとつの原子単位。クリックでダイアログを開き、テキストと yomi 属性を編集
  - `img`：chip として `[🖼 alt]` 表示。クリックで src/alt を編集
- ビルドターゲット：Mac / Windows
